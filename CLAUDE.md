## Architecture Overview

Angular 21 SPA for football betting analysis, focused on draw (empate) streak tracking across 100+ international leagues. All components use the **standalone pattern** — no NgModules anywhere.

### Route Structure

```
/ → redirects to /hot-list
/hot-list     — main table showing draw stats for all leagues simultaneously
/liga         — detailed view for a single league (tabs: results, upcoming, historical, analytics)
/dashboard    — global histogram overview
```

`LayoutComponent` wraps all three routes, providing the navbar and league-selector sidebar.

### Data Flow

**Static data** lives in `public/Ligas/[LeagueName].json` — one file per league with match results and schedules. These are the source of truth for historical analysis.

**Pre-computed streaks** are in `public/historico/[LeagueName].json`, generated by running `node scripts/gen_historico.mjs`. Run this script after updating a league's JSON file.

**Live data** is fetched at runtime from two external APIs:
- Livescore (`/v1/api/*`) — match results
- Sofascore (`/api/sofascore/*`) — odds and additional stats

In dev, `proxy.conf.js` handles these routes with spoofed browser headers. In production (Vercel), `/api/sofascore/[...path].js` is a serverless function that proxies Sofascore.

### Service Layer (core/services/)

| Service | Role |
|---|---|
| `LigasService` | Registry of all league configs (name, API path, Sofascore ID). The single source of league metadata. |
| `HotListService` | Fetches and aggregates data across all leagues for the hot-list view. Most complex service. |
| `LigaDataService` | Fetches detailed data for one selected league. |
| `EmailHotListService` | Sends analysis summaries via EmailJS. |
| `PdfHotListService` | Exports current view as PDF via html2pdf.js. |
| `SchedulerHotListService` | Scheduling/cron utilities. |
| `ThemeService` | Light/dark theme toggle. |

Components inject services with `inject()` (not constructor injection).

### State Management

Angular **signals** are used for reactive state — not RxJS subjects or NgRx. RxJS is used only for HTTP calls.

### Adding a New League

1. Add a JSON file to `public/Ligas/[LeagueName].json` with match data
2. Register it in `LigasService` with name, API path, and Sofascore ID
3. Run `node scripts/gen_historico.mjs` to generate `public/historico/[LeagueName].json`

### Key Interfaces

Defined in `src/app/core/interfaces/results.interface.ts`: `Liga`, `Event`, `HotCheck`, and related types. Check these before adding new properties to components.

### Styling

SCSS + Bootstrap 5.3.8. Global styles in `src/styles.scss`. Component-scoped SCSS in each component's `.scss` file. Angular Material is available but used minimally.

### Deployment

Vercel reads `vercel.json` for rewrites. The only serverless function is `api/sofascore/[...path].js`. Environment variables for API keys (EmailJS) must be set in Vercel project settings and in `src/environments/`.
