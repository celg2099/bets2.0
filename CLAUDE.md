## Architecture Overview

Angular 21 SPA for football betting analysis, focused on draw (empate) streak tracking across 100+ international leagues. All components use the **standalone pattern** — no NgModules anywhere.

### Route Structure

```
/ → redirects to /hot-list
/hot-list     — main table showing draw stats for all leagues simultaneously
/liga         — detailed view for a single league (tabs: results, upcoming, historical, analytics)
/dashboard    — global histogram overview
```

`LayoutComponent` wraps all three routes, providing the navbar and league-selector sidebar.

### Data Flow

**Static data** lives in `public/Ligas/[LeagueName].json` — one file per league. Structure:
```json
{ "descripcion": "...", "fuente": "livescore|sofascore", "tournamentId": "...",
  "temporadas": [{ "nombre": "...", "año": 2024, "seasonId": "...",
    "jornadas": [{ "id": "...", "fechaHora": "DD/MM/YYYY HH:mm",
      "local": "...", "visitante": "...", "estado": "FT|NS|Postp.",
      "goles": [], "resultado": { "golesLocal": 1, "golesVisitante": 0 } }] }] }
```

**Pre-computed streaks** are in `public/historico/[LeagueName].json`, generated by:
```bash
node scripts/gen_historico.mjs LeagueName   # LeagueName must match the file in public/Ligas/
```
Run this after updating a league's JSON. Output structure: `{ nombrePublico, temporadas: [{ temporada, conteos[] }] }` where `conteos` is the array of match-count gaps between draws each season.

**Live data** is fetched at runtime from external APIs via three proxied routes:
- `/v1/api/*` → `https://prod-public-api.livescore.com` (Livescore results)
- `/en/football/*` → `https://www.livescore.com/en/football/` (Livescore web content)
- `/api/sofascore/*` → `https://www.sofascore.com/api/v1` (Sofascore odds/stats)

All proxy routes spoof browser headers (User-Agent, Origin, Referer). In dev, `proxy.conf.js` handles all three. In production (Vercel), `vercel.json` handles the Livescore routes and `api/sofascore/[...path].js` is a serverless function for Sofascore.

### Data Sources per League

Each league in `LigasService` uses either Livescore or Sofascore as its live source:
- If `sofascoreId` is set → uses Sofascore API
- Otherwise → uses Livescore API via `archivoLigas` path

The `enable` flag (1/0) controls whether a league appears in the hot-list.

### Service Layer (core/services/)

| Service | Role |
|---|---|
| `LigasService` | Registry of all ~140 league configs. Key methods: `tieneHistorico()`, `tieneAmbosArchivos()` (used to show ✓ in email). |
| `HotListService` | Fetches all leagues in parallel (`mergeMap(3)` — 3 concurrent), computes draw streaks (`conteo`), percentages, and max records. Most complex service. |
| `LigaDataService` | Fetches and processes data for one selected league. Supports both API sources. |
| `EmailHotListService` | Builds HTML report with 4 sections (Principales, Máximo Conteo, HOT, Histórico Acumulado) and sends via EmailJS. |
| `PdfHotListService` | Exports current view as PDF via html2pdf.js. Filename: `reporte_YYYY-MM-DD_HHmmss.pdf`. |
| `SchedulerHotListService` | Fires at 6:00, 11:00, 16:00, 21:00 UTC daily using RxJS `timer()`+`merge()`. Toggle via signal. |
| `ThemeService` | Light/dark toggle; persists to localStorage, respects `prefers-color-scheme`. Applied via `data-theme` on document root. |

Components inject services with `inject()` (not constructor injection).

### State Management

Angular **signals** are used for reactive state — `signal()`, `computed()`, `effect()`. RxJS is used only for HTTP calls and scheduling. No RxJS subjects or NgRx.

### Key Interfaces

Defined in `src/app/core/interfaces/results.interface.ts`: `Liga`, `Event`, `HotCheck`, `LigaHomologada`, and Sofascore-specific shapes (`SofascoreEvent`, `SofascoreEventsResponse`, etc.). Check these before adding new properties.

`EpsStatus` enum: `'FT'` (Finished), `'NS'` (Not Started), `'Postp.'` (Postponed).

### Adding a New League

1. Add `public/Ligas/[LeagueName].json` with match data
2. Register in `LigasService`: `{ archivoLigas, sofascoreId, enable, historico, ... }`
3. Run `node scripts/gen_historico.mjs LeagueName`

### Styling

SCSS + Bootstrap 5.3.8. Global styles in `src/styles.scss`. Component-scoped SCSS per component. Angular Material is available but used minimally.

### Deployment

Vercel reads `vercel.json` for rewrites. EmailJS keys (`serviceId`, `templateId`, `publicKey`, `destinatario`) must be set both in Vercel project settings and in `src/environments/environment.ts`.
